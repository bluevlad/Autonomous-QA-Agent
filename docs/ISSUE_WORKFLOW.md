# 이슈 워크플로우

테스트 중 발견된 버그 및 개선사항을 관리하는 워크플로우입니다.

## 이슈 유형

### 1. 버그 (Bug)
- 기능이 예상대로 동작하지 않는 경우
- 에러가 발생하는 경우
- UI가 깨지는 경우

### 2. 개선사항 (Enhancement)
- UX 개선이 필요한 경우
- 기능 추가가 필요한 경우
- 성능 개선이 필요한 경우

## 심각도 분류

| 심각도 | 설명 | 예시 |
|--------|------|------|
| **High** | 핵심 기능 장애, 데이터 손실 위험 | API 500 에러, 저장 실패 |
| **Medium** | 기능은 동작하나 불편함 | 잘못된 메시지 표시, 유효성 검사 누락 |
| **Low** | 사소한 문제, 미관상 이슈 | 오타, 스타일 불일치 |

## 이슈 등록 프로세스

### 1단계: 이슈 발견 및 기록

테스트 중 이슈 발견 시 `projects/[project]/issues/` 폴더에 마크다운 파일로 기록합니다.

```markdown
# ISSUE_XXX: 이슈 제목

## 유형
Bug / Enhancement

## 심각도
High / Medium / Low

## 설명
이슈에 대한 상세 설명

## 재현 단계 (버그인 경우)
1. 단계 1
2. 단계 2
3. 단계 3

## 기대 동작
예상되는 정상 동작

## 실제 동작
현재 발생하는 문제

## 스크린샷
(필요한 경우)

## 관련 파일
- 파일 경로:라인번호
```

### 2단계: GitHub Issue 등록

GitHub CLI를 사용하여 해당 프로젝트의 저장소에 이슈를 등록합니다.

```bash
# 버그 등록
gh issue create \
  --repo owner/repo \
  --title "[Bug] 이슈 제목" \
  --body "이슈 내용" \
  --label "bug"

# 개선사항 등록
gh issue create \
  --repo owner/repo \
  --title "[Enhancement] 이슈 제목" \
  --body "이슈 내용" \
  --label "enhancement"
```

### 3단계: 테스트 케이스 연결

발견된 이슈에 대한 테스트 케이스를 작성하거나 기존 테스트에 태그를 추가합니다.

```typescript
test('[BUG-#123] 특정 조건에서 에러 발생', async ({ page }) => {
  // 버그 재현 테스트
  // 버그 수정 후 이 테스트가 통과해야 함
});
```

### 4단계: 추적 및 업데이트

- 이슈가 수정되면 테스트 실행하여 확인
- 테스트 통과 시 이슈 close
- TEST_README.md 업데이트

## 이슈 템플릿

### 버그 템플릿

```markdown
## 설명
[버그에 대한 간단한 설명]

## 심각도
**[High/Medium/Low]**

## 재현 단계
1. [첫 번째 단계]
2. [두 번째 단계]
3. [문제 발생]

## 기대 동작
[정상적으로 동작해야 하는 방식]

## 실제 동작
[현재 발생하는 문제]

## 환경
- Browser: Chrome
- OS: Windows 11

## 관련 파일
- `path/to/file.ts:123`

## 스크린샷
[필요한 경우 첨부]
```

### 개선사항 템플릿

```markdown
## 설명
[개선이 필요한 사항에 대한 설명]

## 우선순위
**[High/Medium/Low]**

## 현재 동작
[현재 어떻게 동작하는지]

## 제안 개선안
[어떻게 개선되어야 하는지]

## 기대 효과
- [효과 1]
- [효과 2]

## 관련 파일
- `path/to/file.ts`
```

## 자동화 스크립트

```bash
# 모든 로컬 이슈를 GitHub에 일괄 등록
npm run issues:create

# 특정 프로젝트의 이슈만 등록
npm run issues:create -- --project=hopenvision
```
